# 字段抽取说明

## 当前能力

这个版本把财务字段抽取从 parser 内部的临时正则，升级成了一个可复用的轻量模块：

- 基于 alias 字典匹配中英文字段名
- 基于规则解析数值、百分比、币种和常见数量级
- 返回证据片段 `source_text`
- 在 OCR 场景下尽量保留页码信息
- 结果中携带 `canonical_code` 和解析元数据，方便后续标准化与排错

当前支持的核心字段：

- `revenue`
- `net_profit`
- `operating_cashflow`
- `free_cash_flow`
- `gross_margin`
- `roe`
- `debt_ratio`
- `revenue_growth`

## 支持的轻量单位/倍数

当前优先覆盖财报里最常见、最容易解释的一批表达：

- 百分比：`%`
- 中文数量级：`亿`、`亿元`、`万`、`万元`、`百万元`
- 英文数量级：`million`、`billion`、`thousand`、`mn`、`bn`
- 币种线索：`RMB`、`CNY`、`USD`、`HKD`、`人民币`、`美元`、`港元`
- 简单全局单位提示：如 `单位：亿元人民币`

解析后会尽量把数值归一到绝对值，例如：

- `12.5 亿` -> `1250000000.0`
- `USD 320 million` -> `320000000.0`
- `44.5%` -> `44.5`

## 输出结构

每个候选字段会尽量包含：

- `name`: 命中的原始 alias
- `canonical_code`: 统一字段编码
- `value`: 已按倍数处理后的数值
- `unit`: `%` 或币种代码
- `source_text`: 原始证据片段
- `page`: OCR 页码（如果可用）
- `metadata`: 命中 alias、原始值、原始单位、倍数等调试信息

## 已知限制

这里不声称已经达到生产级抽取质量。当前仍然有明显边界：

- 主要依赖行级文本，不是版面理解或表格结构恢复
- 对复杂表格、跨行字段、脚注、同比/环比混杂表达支持有限
- 多币种统一换算没有做
- 没有时间维度拆解，例如 Q1/Q2/H1/FY 的严格归因
- 没有做 OCR 错字纠正、表格单元格对齐或置信度排序

## 为什么仍然有用

尽管是启发式实现，它已经比之前的 parser 内嵌正则更适合真实财报文本：

- 可解释：规则和 alias 清晰可查
- 可测试：单位、倍数、OCR 页文本都能单元测试
- 可复用：text / PDF text / OCR 三条路径统一调用
- 可降级：缺失 OCR 或 PDF 依赖时，仍然能对已有文本做抽取
